## 项目结构

```
paly/
├── app/                    # Next.js 页面
│   ├── page.tsx           # 页面1: 战斗页面
│   ├── inventory/
│   │   └── page.tsx       # 页面2: 装备和背包页面
│   ├── skills/
│   │   └── page.tsx       # 页面3: 技能页面
│   ├── layout.tsx         # 根布局
│   └── globals.css        # 全局样式
├── components/            # 组件
│   └── Navigation.tsx     # 底部导航栏
├── store/                 # 状态管理
│   └── gameStore.ts       # Zustand 全局状态
├── types/                 # TypeScript 类型定义
│   └── game.ts            # 游戏数据类型
└── package.json           # 项目配置
```

## 核心功能

### 1. 战斗系统 (app/page.tsx)
- 回合制战斗机制
- 随机生成怪物（史莱姆、哥布林、骷髅）
- 普通攻击 + 4个技能攻击
- 实时战斗日志
- 击败怪物获得经验和金币

### 2. 装备系统 (app/inventory/page.tsx)
- 5个装备槽位：头盔、武器、护甲、靴子、饰品
- 背包系统，支持按类型筛选
- 装备/卸下功能
- 装备属性加成（攻击、防御）

### 3. 技能系统 (app/skills/page.tsx)
- 4个预制技能：重击、治疗、狂暴、防御
- 使用金币升级技能（最高5级）
- 技能效果随等级提升

### 4. 状态管理 (store/gameStore.ts)
使用 Zustand 实现全局状态管理，三个页面共享数据：
- 角色属性（HP、攻击、防御、金币等）
- 装备和背包数据
- 技能数据
- 战斗状态

## 页面联动效果

1. **战斗页 → 装备页**: 战斗中获得的金币可以在装备页使用
2. **装备页 → 战斗页**: 装备的武器和护甲会提升战斗中的攻击和防御
3. **战斗页 → 技能页**: 战斗获得的金币用于升级技能
4. **技能页 → 战斗页**: 升级后的技能在战斗中效果更强
